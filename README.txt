# Ковыляев Роман ИТ-4 Лабораторная работа №4 — Обобщенные типы

## Структура лабораторной работы

# Задание 1. Обобщенные типы

## Задача 1. Обобщенная коробка

### Текст задачи

Создать сущность **Коробка**, которая может хранить один произвольный объект в один момент времени. Объект можно поместить в коробку и получить из нее. Если объект забирают, ссылка на него должна обнуляться. Если попытаться положить новый объект в непустую коробку, должно выбрасываться исключение. Также коробка должна уметь проверять, заполнена ли она. Все методы должны работать с тем типом данных, который был указан при создании объекта.

По условию необходимо создать коробку для целого числа, поместить туда значение `3`, передать коробку в метод, извлечь значение и вывести его на экран.

### Алгоритм решения

1. Создан обобщенный класс `Box<T>`.
2. Добавлено поле для хранения одного объекта типа `T`.
3. Реализованы методы:

   * помещения значения в коробку;
   * извлечения значения из коробки;
   * проверки заполненности.
4. При попытке положить значение в уже заполненную коробку генерируется исключение.
5. После извлечения объекта ссылка внутри коробки очищается.
6. В `main` создается объект `Box<Integer>`, в него помещается число `3`, затем значение извлекается и выводится на экран.

## Задача 3. Сравнимое

### Текст задачи

Создать ссылочный тип **Сравнимое**, который гарантирует наличие метода `сравнить`. Этот метод должен:

* называться `сравнить`;
* принимать объект;
* позволять изменять тип принимаемого объекта без изменения самого сравнимого типа;
* возвращать целое число.

### Алгоритм решения

1. Создан обобщенный интерфейс, аналогичный пользовательскому `Comparable<T>`.
2. В интерфейсе объявлен метод сравнения, принимающий объект заданного типа и возвращающий `int`.
3. Такой подход позволяет задавать тип сравниваемого объекта через параметр типа.
4. Интерфейс далее используется в последующих задачах, где требуется сравнение объектов.

# Задание 2. Параметризация. Маски типов и их ограничения

## Задача 2. Поиск максимума

### Текст задачи

Создать метод, который принимает набор **Коробок** из задачи 1 и возвращает максимальное из их значений в формате `double`. Коробки могут быть параметризованы любыми числовыми типами.

### Алгоритм решения

1. Используется обобщенный метод с ограничением по типу `Number`.
2. Метод принимает набор коробок, внутри которых хранятся числа разных типов.
3. Значения из коробок извлекаются и приводятся к `double` через методы класса `Number`.
4. Среди всех значений ищется максимальное.
5. Результат возвращается как `double`.
6. В `main` демонстрируется работа метода на коробках с разными числовыми типами, например `Integer`, `Double`, `Long`.

# Задание 3. Обобщенные методы. Автовывод типа

## Задача 1. Функция

### Текст задачи

Разработать метод, который принимает список значений типа `T` и объект с единственным методом `apply`. Этот метод нужно применить к каждому элементу списка и вернуть новый список значений типа `P`. Типы `T` и `P` могут как совпадать, так и различаться.

Метод требуется использовать в трех случаях:

1. Преобразовать список строк в список их длин.
2. Преобразовать список чисел так, чтобы отрицательные стали положительными.
3. Преобразовать список массивов целых чисел в список максимальных значений этих массивов.

### Алгоритм решения

1. Создан обобщенный метод преобразования списка.
2. Метод принимает:

   * исходный список `List<T>`;
   * функциональный объект, выполняющий преобразование `T -> P`.
3. Для каждого элемента исходного списка вызывается `apply`.
4. Результаты сохраняются в новый список `List<P>`.
5. В `main` метод демонстрируется на строках, числах и массивах целых чисел.

## Задача 2. Фильтр

### Текст задачи

Разработать метод, который принимает список значений типа `T` и объект с единственным методом `test`, принимающим `T` и возвращающим `boolean`. Метод должен вернуть новый список, из которого удалены все элементы, не прошедшие проверку.

Метод требуется использовать в трех случаях:

1. Оставить только строки длиной не менее трех символов.
2. Отфильтровать положительные элементы списка чисел.
3. Оставить только те массивы целых чисел, в которых нет ни одного положительного элемента.

### Алгоритм решения

1. Создан обобщенный метод фильтрации.
2. Метод принимает исходный список и условие проверки.
3. Каждый элемент проверяется вызовом `test`.
4. Если условие возвращает `true`, элемент добавляется в результирующий список.
5. В `main` показаны все три сценария использования.

## Задача 3. Сокращение

### Текст задачи

Разработать метод, который принимает список значений типа `T` и способ сведения списка к одному значению типа `T`. Метод должен вернуть итоговый результат.

Метод требуется использовать в трех случаях:

1. Объединить список строк в одну большую строку.
2. Найти сумму списка целых чисел.
3. Для списка списков целых чисел получить общее количество элементов.

Дополнительно необходимо изменить метод так, чтобы он гарантированно не возвращал `null` и не выбрасывал ошибку, если исходный список пуст.

### Алгоритм решения

1. Создан обобщенный метод свертки списка.
2. Метод принимает список и правило объединения элементов.
3. Элементы последовательно сводятся к одному результату.
4. Для пустого списка предусмотрена безопасная обработка:

   * возвращается заранее определенное значение по умолчанию, либо
   * используется безопасная стратегия без выброса исключения.
5. В `main` демонстрируется объединение строк, суммирование чисел и подсчет общего количества элементов во вложенных списках.

## Задача 4. Коллекционирование

### Текст задачи

Разработать метод, который возвращает коллекцию типа `P` со значениями типа `T`. Метод принимает:

1. список исходных значений;
2. способ создания результирующей коллекции;
3. способ добавления значений исходного списка в результирующую коллекцию.

Метод требуется использовать в трех случаях:

1. Разбить числа на два подсписка: положительные и отрицательные.
2. Разбить строки на подсписки по одинаковой длине.
3. Получить набор строк без повторяющихся значений.

### Алгоритм решения

1. Создан универсальный обобщенный метод для сборки коллекции.
2. Метод принимает исходный список, фабрику создания коллекции и правило добавления элементов.
3. В зависимости от переданных параметров можно собирать разные типы коллекций.
4. В `main` показаны примеры:

   * группировка чисел по знаку;
   * группировка строк по длине;
   * создание множества без дубликатов.
